This page documents TikiDB, a database processing system for Wireless Sensor Networks. TikiDB was highly inspired by TinyDB (http://telegraph.cs.berkeley.edu/tinydb/) and implemented for the Contiki OS (http://www.contiki-os.org/).

[edit] Installation

1. Check out the source: 

svn co https://www.dvs.tu-darmstadt.de/svn/projects/WSN/TikiDB/trunk/ src

2. Create symbolic links from your Contiki installation: 

cd $CONTIKI/apps
ln -s src/scopes scopes-tikidb
ln -s src/tikidb tikidb

3. Compile applications:

3.1 Scopes tester: 

cd $CONTIKI/apps/scopes-tikidb
make

3.2 TikiDB tester: 

cd $CONTIKI/apps/tikidb
make

3.3 TikiDB shell: 

cd src/tikidb-shell
make

[edit] Sensors

Since the Z1s are only equipped with an accelerometer (not used by us) and a temperature sensor, other sensors were integrated. The files for each sensor can then be found under contiki-2.x/platform/z1/dev/<sensorname>-sensor.{c|h}
There is an additional new file called z1-sensors.{c|h} which has the duty to configure the A/D converter for each single sensor, but its not necessary to know its content.
Additionally you have to update the Z1 makefile, too. You can find it under contiki-2.x/platform/z1/Makefile.z1.
The latest files are in the SVN repository under "solarhaus/sensing".

[edit] Light Sensor

Datasheet: SHARP - GA1A2S100LY
The light sensor has been soldered and the concerning ADC procedure is implemented. It is equipped with a 470 ohm resistor and measures the light sensitivity in lux.
The sensor is connected to AD channel 7.
To use this sensor in Contiki, the following commands are needed (this is similar to the TelosB node): 

 #include "dev/light-sensor.h"
 ...
 light_sensor.configure(SENSORS_ACTIVE,1);
 ...
 static uint16_t light_value = light_sensor.value(LIGHT_SENSOR);


Unlike the TelosB node, we have only one light sensor.

[edit] Humidity Sensor

Datasheet: HONEYWELL - HIH-5030-001
The sensor measures the relative humidity.
As recommended the implementation is equipped with a 65 kOhm resistor.
This resistor is connected between output and ground. The sensor itself can be read out from AD channel 1.
This is how to use it: 

 #include "dev/humidity-sensor.h"
 ...
 humidity_sensor.configure(SENSORS_ACTIVE,1);
 ...
 static int16_t humidity_raw, humidity_value;
 humidity_sensor.configure(SENSORS_ACTIVE,1);
 humidity_raw = humidity_sensor.value(0);
 /*The sensor data will be transmitted as a raw ADC value, so there is a conversion to Celsius necessary:*/
 humidity_value = ((humidity_raw >> 5) - 15);

[edit] Temperature Sensor

Datasheet: Texas Instruments TMP102
The temperature sensor is already implemented on the Z1 node and procedure to read the values is given too: 

 #include "dev/tmp102.h"
 ...
 tmp102_init();
 ...
 raw = tmp102_read_temp_raw();
 if (raw < 0) { // Perform 2C's if sensor returned negative data
   absraw = (raw ^ 0xFFFF) + 1;
   sign = -1;
 }
 tempint  = (absraw >> 8) * sign - 5;
 tempfrac = ((absraw>>4) % 16) * 625; // Info in 1/10000 of degree

[edit] CO2 Sensor Station

The CO2 sensor is included in a sensor station called Voltcraft CO-50. Besides CO2, there is a humidity and a temperature sensor inside.

The driver sensorstation.{c|h} reads a struct, containing all sensor values.

The sensorstation sends the values via an IÂ²C channel to an intermediate PIC microcontroller. The PIC itself parses the information and sends it via UART to the Z1 sensor node in a well understandable form.

This is how it could look like: 

 #include "dev/sensorstation.h"
 ...
 sensorstation_init();
 ...
 struct sensor_values _sensor_values;
 _sensor_values = sensorstation_read();
 printf("CO2: %d  Humidity: %d  Temperature: %d\n",_sensor_values.co2, _sensor_values.humidity, _sensor_values.temperature);
